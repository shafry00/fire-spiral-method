# Fire Spiral Method: Amati → Praktekkan → Integrasikan → ∞

**A 3-phase learning framework built from lived experience, not textbooks.**

> *You don't need to understand before you act. Act first. Understanding follows — in deepening spirals.*

---

## The Problem

Most learning advice tells you to start with theory: *read the docs, watch the tutorial, understand the concepts, then practice.*

That's backwards.

When you start with theory, you have no context. You don't know which concepts matter. You can't tell signal from noise. You get stuck preparing forever without ever doing anything real.

The Fire Spiral Method flips this: **action first, concepts later.** Every spiral digs one layer deeper into the bottleneck that actually stopped you — not the bottleneck you imagined before you started.

## The Three Phases

```
                    ↺
               ┌────────────┐
               │INTEGRASIKAN│  ← Gap → Triage → Resource → Loop
               │ (Integrate)│
               └──┬─────────┘
                  │ new insight emerges
               ┌──┴─────────┐
               │PRAKTEKKAN │  ← Targeted action, deliberate practice
               │ (Practice) │
               └──┬─────────┘
                  │ real gap surfaces
               ┌──┴─────────┐
               │  AMATI     │  ← Trigger, problem framing
               │ (Observe)  │
               └────────────┘
                    ↻
```

### 1. AMATI — Observe

You feel curiosity. Or urgency. Or both. Frame the problem:

- **What's the real problem?** (1-2 sentences)
- **Why does it matter?** (consequence of not solving it)
- **Constraint Framing**: Target → Minimum success → Cost of failure

### 2. PRAKTEKKAN — Practice

Direct action. No preparation, no theory. Try, break, experiment, fail messily.

Capture every gap: what you *thought* would happen vs what *actually* happened.

**Gap Log format (3 lines):**
```
Thought: [expectation before acting]
Reality: [what actually happened]
Insight: [the gap between expectation and reality]
```

### 3. INTEGRASIKAN — Integrate

Bridge between spirals. Review gaps, decide what to do next:

- **Go** — priority, tackle in next spiral
- **Review** — important but not urgent, shelve
- **Kill** — discard, turned out irrelevant

Then: deeper spiral on same topic, pick up a dormant topic, or start something new.

## Drive Score: One Path, Adaptive Intensity

Not everything you learn deserves full depth. The Drive Score determines how deep you go:

| Dimension | Question | Scale |
|-----------|----------|-------|
| **Interest** | How curious are you? | 0 (don't care) – 10 (obsessed) |
| **Freedom** | Tools / access / time? | 0 (none) – 10 (everything) |
| **Pressure** | External deadline or demand? | 0 (relaxed) – 10 (tomorrow) |
| **Gap** | How deep the target? | 1 (surface) – 10 (mastery) |

### Quick Calculation

```
WANT = (Interest + Freedom) / 2
PUSH = (Pressure + (10 - Gap)) / 2
```

### Zone Map

```
PUSH ↑
10  │  SURVIVAL (≤0.40)     HYBRID (0.41–0.65)
    │  compressed, fast      selective depth
 7  │ ─────────────────────────────────
    │  START                 CURIOSITY (≥0.66)
 3  │  too early to tell     full spiral, no exit
    │ ─────────────────────────────────
 0  └────────────────────────────────→ WANT
    0        5        7        10
```

### Time Box Per Zone

| Zone | AMATI | PRAKTEKKAN | INTEGRASIKAN |
|------|-------|-----------|-------------|
| **Survival** (≤0.40) | 5m | 3 × 25m | 5m |
| **Hybrid** (0.41–0.65) | 15m | 3-4 × 45m | 15m |
| **Curiosity** (≥0.66) | 30m | 4 × 45m | 30m |

**One path, adaptive intensity.** Same 3-phase structure — but the depth of tools and reflection scales with your Drive Score.

## Proof Anchor: Can't Lie to Reality

> *"You can lie to yourself. You can't lie to official documentation, experiment results, or primary data."*

At 3 decision points, anchor your claims to external reality:

| Gate | When | Ask |
|------|------|-----|
| **Gate 1** | AMATI → PRAKTEKKAN | Is your Problem Statement referenced to something real? Or all "I think" and "it seems"? |
| **Gate 2** | Gap Log → INTEGRASIKAN | Is "Reality" backed by concrete evidence? Not "turns out it's harder" but "error 404, route `/x`, 3 attempts" |
| **Gate 3** | Triage Output | Does every "Go" have external grounding? Every "Kill" have evidence of irrelevance? |

**3-Source Challenge**: 1 source is baseline. If it contradicts your claim → find 2 more. 2/3 against → claim is wrong.

**Witness Rule** (for non-technical domains): No compiler to tell you if you're right? 1 independent observer + 3 testers + end-to-end documentation = sufficient validation.

## Golden Rules

- **Action first, concepts later.** Theory arrives when you have context.
- **Go = 1 step, 1 result.** Not 1 comprehensive project.
- **Gaps are data, not failure.** Every "this didn't work" is the path.
- **Narrow when stuck, don't add effort.** Smaller scope > more time.
- **Confidence before comprehension.** First meeting: "I can do this." Understanding comes later.
- **Constraint Framing is mandatory** in Curiosity Zone. Prevents over-engineering.
- **Spiral next always starts from current bottleneck.**

## Why This Is New — Compared to Existing Learning Theories

Existing frameworks were built in classroom eras — teacher-centered, curriculum-driven, externally validated. The Fire Spiral Method was built in the self-directed era. Here's what it does differently.

### Six Innovations

**1. Action-First Inversion.** Kolb starts with "Abstract Conceptualization." Bloom begins with "Remember." Vygotsky's ZPD assumes a teacher provides the concept first. Fire Spiral inverts this: **PRAKTEKKAN begins with raw action, before any theory.** Concepts enter only at INTEGRASIKAN — and only when concrete gaps make the theory meaningful.

**2. No Exit Condition — Infinite Deepening.** Every major framework terminates: Bloom's mastery learning ends at 80-90%. The Feynman Technique ends when you "explain it simply." Fire Spiral never terminates. Each pass through the spiral reveals **deeper, more fundamental gaps** beneath the surface ones. Learning is treated as infinite deepening, not milestone completion.

**3. Problem-Deepening Over Problem-Solving (90/10 Ratio).** Design thinking rushes to prototype. PBL rushes to deliverable. Tutoring rushes to "getting it right." Fire Spiral deliberately **holds the gap open** — each cycle aims to reveal a more fundamental version of the problem, not to solve the surface one.

**4. Built-In Validation — No Teacher Required.** Existing frameworks depend on external validation: teacher grading, peer review, mentor feedback. Fire Spiral embeds validation into the framework itself: **Proof Anchor** (3 falsifiable gates), **Honesty Check** (5 self-deception signals), and **Witness Rule** (for domains without objective verifiers).

**5. Zone-Adaptive Scaffolding.** Drive Score — a 4-dimensional metric — auto-tunes the framework's intensity. Survival Zone gets compressed fast-exit scaffolding. Curiosity Zone gets the full toolkit. This acknowledges that learning contexts vary; the framework adapts rather than applying uniformly.

**6. Constraint Framing for Intrinsic Learners.** The primary failure mode in self-directed learning is not lack of discipline — it's **drift**. Constraint Framing prevents over-engineering without killing curiosity.

### What It Inherits (With Attribution)

| Concept | From | Adaptation |
|---------|------|------------|
| Doing → Reflecting cycle | Kolb (1984), Dewey (1938) | Action-first inversion; reflection is lightweight |
| Spiral revisiting | Bruner (1960) | Infinite deepening, not curriculum-driven revisiting |
| Explain it simply | Feynman Technique | Integrated into Proof Anchor Gate 2 |
| Learning by making | Papert's Constructionism (1980) | PRAKTEKKAN inherits this directly |
| Double-loop learning | Argyris & Schön (1978) | Embedded in Blank Slate debrief |

### What It Challenges

| Theory | Challenge |
|--------|-----------|
| **Cognitive Load Theory** (Sweller 1988) | PRAKTEKKAN deliberately overloads — the confusion IS the learning signal |
| **ZPD Scaffolding** (Vygotsky) | Learners self-determine when to seek help via People Lens, no MKO required |
| **Mastery Learning** (Bloom 1968) | A topic is never "mastered" — only deepened. Philosophical disagreement. |
